| `src/pages/fleet.astro` | Fleet case — passes description + ogImage + jsonLd |
| `src/pages/asset.astro` | Asset case — passes description + ogImage + jsonLd |

## OG image

- **Default** (`/og.png`): used by the homepage and as a fallback. Designed card — Caveat name, red TV italic, "talks" struck through, "ships" in red, dot grid texture, ghosted S, taking-briefs badge.
- **Case pages**: use their hero screenshots (`/assets/cockpit/cockpit-hero.png` etc.)
- **Source file**: `public/og-card.html` — open in browser at `localhost:4322/og-card.html`, screenshot at 1200×630 with Playwright, copy result to `public/og.png`.

To regenerate `og.png` after editing `og-card.html`:
```bash
# In Claude Code — Playwright MCP
browser_resize(1200, 630)
browser_navigate('http://localhost:4322/og-card.html')
browser_take_screenshot('og-new.png')
# Then copy to public/og.png
```

## Favicon

`public/favicon.svg` — SVG favicon with inline `@font-face` loading Caveat from `/fonts/Caveat[wght].woff2`.

- Shape: dark circle (`#1a1a1c`)
- Letter: "S" in Caveat bold, brand red (`#d8352a`)
- Linked in BaseLayout: `<link rel="icon" type="image/svg+xml" href="/favicon.svg" />`

To change: edit the SVG directly. For PNG fallback (iOS homescreen), add `<link rel="apple-touch-icon" href="/apple-touch-icon.png">` and provide a 180×180 PNG.

## Structured data (JSON-LD)

Every page gets a `Person` schema. Case pages additionally get a `CreativeWork` schema.

**Person schema** (auto-generated in `BaseLayout.astro`):
```json
{
  "@type": "Person",
  "name": "Sivanesh TV",
  "url": "https://sivanesh.tv",
  "jobTitle": "Product Designer",
  "sameAs": [
    "https://www.linkedin.com/in/sivanesh-t-v-product-designer",
    "https://x.com/dunkadamics"
  ]
}
```

**CreativeWork schema** — passed via `jsonLd` prop in each case page. Fields: `name`, `description`, `url`, `author`, `keywords`.

To verify in browser: DevTools → Console → `JSON.parse(document.querySelector('script[type="application/ld+json"]').textContent)`

## Canonical + sitemap

- **Canonical**: auto-built from `Astro.site` + `Astro.url.pathname` in BaseLayout. Points all pages to `https://sivanesh.tv/...` regardless of which domain (Netlify subdomain, etc.) is serving.
- **Sitemap**: generated by `@astrojs/sitemap` at build time. Available at `https://sivanesh.tv/sitemap-index.xml` after deploy.
- **robots.txt**: `public/robots.txt` — allows all crawlers, points to sitemap.

## Accessibility (Lighthouse scores — mobile)

Last audit: **Accessibility 100 · Best Practices 100 · SEO 92\***

\* The SEO 92 is caused by a "Learn more" link in Astro's DevToolbar (dev-only). In production this is 100.

Fixes applied:
- `maximum-scale` changed from `1` to `5` (was blocking pinch-zoom)
- "GOT IT" button background darkened to `#b82b22` (contrast was 3.83:1, needed 4.5:1)
- Nav menu buttons given `role="menuitem"` to match parent `role="menu"`
- `menuToggle` aria-label removed (visible text is the accessible name; `aria-haspopup="menu"` added instead)
- `ContactCard` heading changed from `h3` to `h2` (heading order was broken)

## What to update when...

**Changing your tagline/description**: update `BaseLayout.astro` default + `workbench.ts` masthead `sub` field.

**Adding a new case study**: create the page, pass `description`, `ogImage`, and `jsonLd` to `CaseLayout`. Add the page to this doc's table above.

**Updating the OG card design**: edit `public/og-card.html`, re-screenshot at 1200×630, copy to `public/og.png`.

**Changing social handles**: update `personSchema.sameAs` in `BaseLayout.astro` + `twitter:creator` meta tag.
