**One CSS diff:**

**Polaroid hover transform:**

| | Value |
|---|---|
| **Spec (05-components.md)** | `transform: translateY(-4px) rotate(var(--rotation, 0deg))` |
| **Implementation** | `transform: translateY(-4px) rotate(calc(var(--rotation, 0deg) * 0.7))` |

The implementation reduces rotation toward 0 on hover (multiplying by 0.7). This is actually the *intended* behavior described in the spec prose ("reduces rotation toward 0 … 'picking it up to look'") and in `06-known-gotchas.md` G13. The CSS reference line in 05-components.md appears to be a documentation error — the spec prose and the gotchas doc both describe the 0.7 reduction behavior. **Not a bug, but a doc inconsistency.**
